## New Patterns Discovered

1. **Route53 zones as orphan type**: hiveutil aws-tag-deprovision leaves Route53 zones behind.
   23 truly orphaned zones found (after CallerReference verification). These accumulate over time
   and need a separate cleanup path (not via hiveutil). Added to manifest as `route53_zone` type.
   Always check CallerReference — zone name alone is unreliable:
   - Hive CallerRef: `<infra_id>-<random>` (check infra_id against live list)
   - Terraform CallerRef: `terraform-<timestamp>` (fips/non-Hive zones; check cluster name in live CDs)

2. **ROSA HCP velero buckets**: 44 managed-velero-backups-* buckets, all for ROSA HCP clusters.
   42 orphaned. Tagged with `velero.io/infrastructureName` = ROSA infra ID. These are HIGH confidence
   when last write > 7d ago and no active cluster found.

3. **hub-4xx cluster remnants**: hub-type clusters (app-aws-east2-415, app-aws-east1-420,
   app-aws-west2-417) leave VPC/subnet/NAT networking resources behind but their IAM profiles
   persist. IAM is the reliable age signal.

4. **Zone name ≠ cluster identity**: Zone `app-prow-small-aws-421-west2-pthps` belongs to live CD
   `app-prow-small-aws-42-sptgz`. The zone includes pool version/region context (`421-west2`) in the
   name, not the CD infra ID. Never trust zone name suffix as infra ID without CallerReference check.
